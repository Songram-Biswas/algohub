    💡 Secret tricks

    • Count the loops, not the lines. One pass over n elements with O(1) work per element gives O(n); nested loops over the same input usually multiply.
    • Hash set/map operations are O(1) average. A loop doing one lookup or insert per iteration stays O(n) overall — the hash structure doesn't add a factor.
    • Space tracks "extra" memory, not the input. Ask what new structure you're building and how big it can get.
    • Two pointers / sliding window ≠ O(n²). Even though it looks like nested loops, each pointer only ever moves forward — total work stays O(n).

    Walk the array once, tracking every value you've already seen in a hash set. The moment you encounter a value already in the set, you've found a duplicate.

    1. For each number, check if it's already in seen.
    2. If it is, return True immediately.
    3. Otherwise, add it to seen and continue.
    4. If the loop finishes without a hit, return False.
    from typing import List

class Solution:
    def containsDuplicate(self, nums: List[int]) -> bool:
        seen = set()

        for num in nums:
            if num in seen:
                return True
            seen.add(num)

        return False

    Time

    O(n)

    Single pass; each hash set lookup/insert is O(1) average.

    Space

    O(n)

    In the worst case (no duplicates) the set holds all n elements.

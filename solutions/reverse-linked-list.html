206. Reverse Linked List

Easy
Solve on LeetCode

Description

Given the head of a singly linked list, reverse the list in-place and return the new head.

Constraints

  • • The number of nodes in the list is the range [0, 5000].
  • • -5000 <= Node.val <= 5000
  • • Must be done in-place

Examples

Example 1:

Input: head = [1,2,3,4,5]
Output: [5,4,3,2,1]

Visual:
Before: 1 → 2 → 3 → 4 → 5 → NULL
After:  5 → 4 → 3 → 2 → 1 → NULL

Example 2:

Input: head = [1,2]
Output: [2,1]

Visual:
Before: 1 → 2 → NULL
After:  2 → 1 → NULL

Example 3:

Input: head = []
Output: []

Visual: Empty list remains empty

Approach & Key Insights

🔴 The Key Insight

To reverse a linked list without extra data structures, we need three pointers:

  • prev — Points to the reversed part
  • curr — Points to the current node being processed
  • next — Saves the next node before we flip

Why Do We Need Three Pointers?

When we execute curr.next = prev, we overwrite the reference to the next unprocessed node. Without saving it first, we lose access to the rest of the list!

❌ WRONG:

curr.next = prev
curr = curr.next

✅ CORRECT:

next = curr.next
curr.next = prev
curr = next

Step-by-Step Process

Input: 1 → 2 → 3 → NULL

Iteration 1: Process node 1

prev = None,  curr = 1,  next = 2
After flip:  prev = 1 → None,  curr = 2 → 3

Iteration 2: Process node 2

prev = 1,  curr = 2,  next = 3
After flip:  prev = 2 → 1,  curr = 3

Iteration 3: Process node 3

prev = 2,  curr = 3,  next = None
After flip:  prev = 3 → 2 → 1,  curr = None

Return: prev

Complexity Analysis

  • Time Complexity: O(n)
  • Space Complexity: O(1)
  • In-place: ✓ Yes

Solution & Code

Python Solution (Iterative)

class Solution:
    def reverseList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        prev = None
        curr = head
        
        while curr:
            next_node = curr.next
            curr.next = prev
            prev = curr
            curr = next_node
        
        return prev

Key Lines Explained

Line 1: next_node = curr.next

We must save the reference to the next node before we flip the pointer. Once we do curr.next = prev, the original forward reference is lost forever.

Line 2: curr.next = prev

This is the core reversal operation. We make the current node point backward instead of forward.

Bonus: Recursive Solution

class Solution:
    def reverseList(self, head: Optional[ListNode]) -> Optional[ListNode]:
        if not head or not head.next:
            return head
        
        new_head = self.reverseList(head.next)
        
        head.next.next = head
        head.next = None
        
        return new_head

Note: The recursive solution is elegant but uses O(n) space for the recursion stack. The iterative solution is preferred because it's in-place with O(1) space.
